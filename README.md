This is the repo for **CROPS Circle**, a peer-curated index of autonomous technologies — tools for coordination and creation free of predation, extraction, censorship, or surveillance by power of all stripes (nation-states, multinational corporations, oligarchs).

CROPS stands for **Censorship Resistant, Open Source, Private, Secure**. The index is maintained by contributors who add, remove, and refine entries over time. If you'd like to help, see [how to contribute](#how-to-contribute).

## What's in the index

The index is organized by category — Hardware, Operating Systems, Communication, Networking & Anonymity, Data & Storage, Identity, Money, Search & Browsing, Development, AI, and Governance — and surfaces a curated short-list under each "What are your technology needs?" answer in the modal.

A tool belongs in the index if it meaningfully meets all four CROPS criteria. We do not list tokens, ad-funded "free" tools, or proprietary SaaS, regardless of how good their UX may be.

## How CROPS Circle is built

- **A single static HTML file.** No build step, no JavaScript, no dependencies.
- **No cookies, no trackers, no analytics.** The site does not know its visitors.
- **Local assets only.** No remote fonts, scripts, or CDN resources.

To preview locally, just open `index.html` in any browser.

## How to contribute

Contributions of any kind are welcome — adding a tool, removing a stale entry, sharpening a description, improving the layout. Before you start, please read [CONTRIBUTING.md](CONTRIBUTING.md) and [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

The short version:

### 1. Submit an issue

Open an issue describing the tool you'd like to add (or remove), which CROPS criteria it meets, and where it belongs.

### 2. Fork the repo

[How to fork a repo](https://help.github.com/en/articles/fork-a-repo).

### 3. Make your changes

Edit `index.html`. The format for each entry — and the two places a tool may appear — is documented in [CONTRIBUTING.md](CONTRIBUTING.md). Open the file in a browser to preview.

### 4. Submit a pull request

Reference the issue in your PR description, e.g. `Adds Tool X [Fixes #42]`.

### 5. Wait for review

Maintainers review every PR. Acceptable PRs are approved and merged.
